## Obsidian Integration

**Configuration:** `nvim.bak.2025.10/lua/plugins/obsidian.lua`

```lua
{
  "epwalsh/obsidian.nvim",
  version = "*",
  lazy = true,
  ft = "markdown",
  dependencies = {
    "nvim-lua/plenary.nvim",
  },
  opts = {
    workspaces = {
      {
        name = "GTD",
        path = "~/Library/Mobile Documents/iCloud~md~obsidian/Documents/Obsidian Cloud",
      },
    },
    notes_subdir = "11 INBOX",
    templates = {
      folder = "99 Utility/Templates",
      date_format = "%Y-%m-%d",
      time_format = "%H:%M",
    },
    disable_frontmatter = true,
    completion = {
      nvim_cmp = true,
      min_chars = 2,
    },
    mappings = {
      ["gf"] = {
        action = function()
          return require("obsidian").util.gf_passthrough()
        end,
        opts = { noremap = false, expr = true, buffer = true },
      },
    },
  },
}
```

## Workflow System

**Configuration:** `nvim.bak.2025.10/lua/config/workflows.lua`

This file contains a comprehensive productivity workflow system:

### Keybindings

| Key | Action | Description |
|-----|--------|-------------|
| `<leader>oo` | Navigate to vault | Opens Obsidian vault root |
| `<leader>on` | Create note | Create new note from template |
| `<leader>ot` | Format title | Format note title |
| `<leader>of` | Find files | Telescope search in notes |
| `<leader>or` | Search content | Live grep in notes |
| `<leader>ok` | Move to review | Move note/dir to zettelkasten |
| `<leader>odd` | Delete note | Delete current note |

### Smart Directory Detection

The workflow includes intelligent directory handling:

```lua
function()
  local current_file = vim.fn.expand("%:t:r")  -- filename without extension
  local parent_dir = vim.fn.expand("%:p:h:t")   -- parent directory name

  if current_file == parent_dir then
    -- Parent directory matches file name
    -- Move the entire directory
  else
    -- Just move the file
  end
end
```

This detects folder structures like:
```
Project-Name/
  Project-Name.md
  attachments/
```

And moves the entire folder rather than just the markdown file.
